
Spring: é uma estrutura java que suporta muitos aplicativos, microsservices e ferramentas de processamento de dados.

Spring core: fornece recursos essenciais, como programação orientada a aspectos.

Spring MVC: serve como uma estrutura web que separa a lógica de negócios, a interface do usuário e o processamento de entrada.

Spring AOP (programação orientada a aspectos): permite o gerenciamento de questões transversais, como registros e tratamento de transações, mantendo-a separada da lógica de negócios. Ele melhora a manutenção do aplicativo porque isola partes do aplicativo, como segurança...

Spring security: fornece uma estrutura personalizável para autentificação e controle de acesso, adicionando recursos de segurança aos aplicativos.

Spring Java Database Connectivity (JDBC): simplifica as interações de banco de dados gerenciando o tratamento de erros.

Spring Web Flow: simplifica a navegação e o gerenciamento de estado em aplicativos web.

Spring Web Services: facilita o desenvolvimento de protocolos de acesso simples ou serviços web SOAP, permitindo uma comunicação segura entre plataformas.

Spring batch: processamento de blocos e agendamento de tarefas.

Documentos REST do Spring: ajudam a documentar a transferência de estado, permitindo que a documentação da API permaneça sincronizada com o código.

Spring session: gerencia as sessões dos usuários. 

Spring boot: acelera o desenvolvimento de aplicativos simplificando a configuração e a implantação.

Spring CLI: permite prototipagem rápida de aplicativos.

Spring Cloud: fornece ferramentas para construir sistemas distribuídos.

Spring WebFlux: para criar aplicativos assíncronos e podem lidar com grandes volumes de solicitações.

Spring Apache Kafka: suporta a criação de sistemas orientados, permitindo um streaming eficiente de mensagem.

Spring Cloud Gateway: gateway de API que encaminha solicitações. 

Beans: são objetos gerenciados pelo contêiner Spring Inversion of Control, eles são fundamentais para um aplicativo e formam os principais componentes que fornece funcionalidade em um aplicativo Spring.

Contexto de aplicação: ele gerencia os Beans e suas dependências durante toda a vida do aplicativo, assim, garante que todos os componentes necessários estejam disponíveis quando necessário.

Inversão de controle: é um princípio fundamental que transfere a responsabilidade de criar e gerenciar dependências do aplicativo para estrutura Spring. Exemplo: no lugar de criar objetos manualmente, como um motor de um carro, Spring fornece os componentes necessários ao inicializar os beans.

Injeção de dependência (DI): é uma técnica que permite que dependências sejam injetadas em objetos em tempo de execução. Exemplo: se um carro precisa de motor para operar, o Spring injeta esse motor no carro quando o carro Bean é inicializado.

Fiação automática: injeta automaticamente as dependências necessárias nos Beans, sem defini-las explicitamente.

Componentes: são classes gerenciadas diretamente pelo contêiner ioC (Inversão de controle). Eles são anotados usanso @component e podem ser registrados automaticamente como beans no contêiner. Componentes podem ser serviços, repositórios... porque eles trabalham juntos para garantir o bom funcioanemnto do aplicativo. 


IntelliJ IDEA: é um ambiente de desenvolvimento integrado desenvolvido pela JetBrains, suportando várias linguagens de programação, com foco em Java. Ela é muito escolhida porque tem: interface amigável, ferramentas de depuração, terminal embutido...


Anotações: servem para ajudar a estrutura a entender como lidar com componentes, gerenciar dependências e configurar aplicativos. Anotações comuns da primavera:

@Component: declara uma classe como um Bean gerenciado pelo Spring, permitindo que ela seja registrada automaticamente no IOC. Exemplo:

@Component
public class BookService{

}

Nesse momento ele fica disponível para injeção de dependência no aplicativo.


@Controller: marca uma classe como WebController permitindo que ela manipule solicitações do HTTP. Tornando responsável por processar solicitações e retornar a visualização adequada. 

@Autowired: permite a injeção automática de dependências, permitindo que o contêiner Spring resolva e injete dependências sem configuração explícita.

@Configuration: marca uma classe como fonte de definições de Bean. Permitindo que o Spring gerencie e instancie Beans.

@Bean: é usada dentro de uma classe @Configuration para definir um método que retorna um objeto gerenciado pelo contêiner Spring.

@RequestMapping: mapeia solicitações HTTP para métodos ou classes específicos em um aplicativo MVC.

@PostMapping: lida com solicitações HTTP POST para adicionar novos registros.

@PathVariable: extrai valores de um caminho URI e os vincula aos parâmetros do método. Exemplo:

public String getBookById(@PathVariable("id") String bookId).

Pathvariable aqui é usada para recuperar um id.

@SpringBootApplication: combina as funcionalidades de definição de beans, configuração automática e verificação de componentes. 

@RequestParam: extrai parâmetros e os vincula aos parâmetros do método. 

@ResponseBody: diz ao Spring que grave o valor de retorno do método diretamente no corpo da resposta HTTP.

@Scope: define o ciclo de vida e a visibilidade de um Bean.

Anotações acima das classes são colocadas para defini-las como um Bean. Eles ajudam a identificar e registrar a classe para injeção de dependência e outras funcionalidades.

Anotações em nível método, como @Bean, são colocadas acima das declarações de método para configurar seu comportamento. Essas anotações informam ao Spring como o método deve ser executado, como definir rotas...

Anotações em nível de campo: são colocadas diretamente acima dos campos para injetar dependência ou valores. Isso permite que o Spring forneça automaticamente objetos necessários.


Maven: é uma poderosa ferramenta de gerenciamento e compressão de projetos usada principalmente em projetos java. Ele fornece uma estrutura para automatizar o processo de criação de software, ajudando os dev a gerenciar dependências, compilar códigos, executar testes... Os principais aspectos Maven são o arquivo POM (Project Object Model), as dependências, o ciclo de vida e os plug-ins.

POM: é o componente mais crítico porque ele contém informações sobre os detalhes de configuração usadas pelo Maven para construir o projeto, suas dependências externas... Eles dependem de bibliotecas externas para funcionarem corretamente. A compreensão dessa estrutura ajuda a gerenciar o código, organizar arquivos...

Repositórios: são os locais onde o Maven armazena os arquivos de biblioteca.

Ciclo de vida: validar, compilar, testas, empacotar, verificar, instalar e implantar.

Plug-ins: para compilar código ou gerar documentação.


Componentes do arquivo POM.xml:

Elemento do projeto: ele define o espaço de nomes XML e o local.

Grupo ID: representa o grupo à qual o projeto pertence e geralmente é estruturado como um nome de domínio.

Artefato ID: é o nome exclusivo do projeto ou módulo.

Dependências: são todas as bibliotecas externas que o projeto usa.

Repositorios: lista os repositórios adicionais onde o Maven deve procurar dependências se elas não forem encontradas no repositório padrão.

Properties: permite definir várias variáveis personalizáveis. 

Profiles: permite definir configurações diferentes para ambientes diferentesa, como desenvolvimento ou produção.


Processo de definição de um projeto:

A primeira etapa é baixar e instalar o JDK.

Em seguida, baixe e instale o Maven.

Crie um projeto Maven através de um comando dado pela plataforma. Esse comando define o ID do grupo e ID do artefato como nome do projeto.

Entenda a estrutura do projeto. Quando um projeto Maven é criado, ele segue uma estrutura padrão que ajuda a organizar os arquivos. 

Dentro de src/main/java são colocados todos os arquivos de código-fonte, incluindo classes de configuração...

A pasta src/main/resources armazena arquivos de configuração como HTML, CSS.

A pasta src/test/java é usada para escrever casos de teste. 

A pasta src/test/resources contém recursos adicionais necessários durante o teste, como arquivo de configuração. 

 Próximo passo é adicionar as dependências necessárias. Abra o arquivo pom.xml e inclua as bibliotecas necessárias do Spring.
 
 Crie um aplicativo Spring básico. Isso envolve a definição de uma classe de configuração, um bean simples e uma classe de aplicativo principal. Dentro do diretório src/main/java/com/example crie uma classe e faça a anotação @Configuration. Essa classe define um Bean usando a anotação @Bean que permite que o Spring gerencie e crie instâncias do Bean.
 
 Agora, escreva uma classe Java que represente o Bean. Essa classe contém um método que executa operações básicas, como imprimir uma mensagem no console.
 
 Desenvolva uma classe principal, recupere o Bean e chame seu método.
 
 Para compilar e executar, use o Maven para criar o projeto executando o comando compile e em seguida, execute a classe principal para iniciar o aplicativo.
 
 
 Spring Initializr: é uma ferramenta que simplifica a configuração aplicativos Spring Boot. Ele fornece uma interface amigável para gerar a estrutura básica dos projetos.
