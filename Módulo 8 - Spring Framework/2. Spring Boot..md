# Spring Boot

**Spring Boot** simplifica o desenvolvimento ao reduzir configurações manuais e permitir que o desenvolvedor foque na lógica do aplicativo.

Ele é construído sobre o Spring Framework e fornece configurações prontas.

---

## Recursos principais

### Configuração automática

Configura automaticamente beans e componentes com base nas dependências do projeto.

---

### Aplicações autônomas

Inclui servidor web embutido, permitindo executar o aplicativo sem servidor externo.

---

### Recursos prontos

Fornece ferramentas integradas que aceleram o desenvolvimento.

---

### Spring Boot CLI

Permite criar e testar aplicações rapidamente via linha de comando.

---

### Spring Initializr

Gera projetos prontos com dependências e estrutura básica.

---

### Servidor incorporado

Dispensa configuração externa.

Servidores suportados:

- Tomcat (padrão)
    
- Jetty
    
- Undertow
    

---

### Actuator

Fornece informações sobre:

- saúde do sistema
    
- métricas
    
- desempenho
    

---

### Configuração externalizada

Permite separar configurações do código.

Exemplo:

server.port=8080

---

# Spring Framework vs Spring Boot

## Spring Framework

Use quando:

- precisa de controle detalhado
    
- aplicações corporativas complexas
    
- manutenção de sistemas legados
    
- migração gradual
    

---

## Spring Boot

Use quando:

- precisa desenvolver rápido
    
- microsserviços
    
- aplicações cloud-native
    
- prototipagem rápida
    
- APIs REST modernas
    

---

- linguagem

# Criando um Projeto Spring Boot

## 1️⃣ Gerar projeto

Use o Spring Initializr.

Inclua:

- Spring Web (para APIs REST)
    

---

## 2️⃣ Criar controlador

@RestController  
public class BookController {  
  
    @GetMapping("/books")  
    public String listBooks() {  
        return "Lista de livros";  
    }  
}

---

## 3️⃣ Criar modelo

public class Book {  
    private String title;  
    private String author;  
  
    public Book(String title, String author){  
        this.title = title;  
        this.author = author;  
    }  
  
    public String getTitle(){ return title; }  
    public void setTitle(String title){ this.title = title; }  
  
    public String getAuthor(){ return author; }  
    public void setAuthor(String author){ this.author = author; }  
}

---

## 4️⃣ Configuração

Arquivo:

application.properties

Exemplo:

server.port=8080

---

## 5️⃣ Executar aplicação

Execute a classe principal.

---

## Logs

Configurar em:

application.properties

Exemplo:

logging.level.root=INFO

# REST (Representational State Transfer)

Permite acessar recursos via web.

Você solicita um recurso sem conhecer a lógica interna.

---

## Componentes REST

### Recursos

Objetos acessíveis via URI.

### Métodos HTTP

- GET
    
- POST
    
- PUT
    
- DELETE
    

### Stateless

Cada requisição contém todas as informações necessárias.

### Formatos

- JSON
    
- XML
    

---

# Requisições HTTP

## GET

Recupera dados.

## POST

Cria novos recursos.

@PostMapping  
@RequestBody

## PUT

Atualiza recursos.

@PutMapping  
@PathVariable

## DELETE

Remove recursos.

---

# Parâmetros de API

### Path

Identifica recurso.

### Query

Filtra dados.

### Body

Envia dados.

### Header

Autenticação e metadados.

---

# Códigos de Status HTTP

### 1xx

Informativo

### 2xx

Sucesso

### 3xx

Redirecionamento

### 4xx

Erro do cliente

### 5xx

Erro do servidor

---

# Versionamento de API

## Métodos

### URI

/api/v1/books

### Query

/books?version=1

### Header

Versão no cabeçalho HTTP.

---

## Versionamento Semântico

Formato:

MAJOR.MINOR.PATCH

- MAJOR → mudanças incompatíveis
    
- MINOR → novos recursos compatíveis
    
- PATCH → correções

# Java Streams (Resumo)

## O que são?

Permitem processar dados de forma eficiente.

---

## Operações intermediárias

- filter()
    
- map()
    
- sorted()
    

---

## Operações finais

- forEach()
    
- collect()
    
- reduce()
    

---

## Funções importantes

✔ filter → filtra elementos  
✔ map → transforma dados  
✔ sorted → ordena  
✔ limit → limita resultados  
✔ collect → agrupa dados  
✔ reduce → reduz a um valor

---

## Streams paralelos

Melhoram desempenho em CPUs multi-core.
