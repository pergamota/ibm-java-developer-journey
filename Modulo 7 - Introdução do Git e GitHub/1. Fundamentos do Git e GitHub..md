
# Termos Básicos do Git e GitHub

## SSH Protocol

É um método para login seguro.

---

## Repositório

Contém as pastas do seu projeto.

---

## Bifurcação (Fork)

É a cópia de um repositório.

---

## Pull Request

É como você solicita que alguém revise e aprove suas alterações antes que elas se tornem definitivas.

---

## Diretório de Trabalho

Contém arquivos e subdiretórios associados a um repositório Git.

---

## Ramificações (Branches)

É uma linha de desenvolvimento que permite trabalhar em correções de forma independente.

---

## Merge (Margem)

Combina mudanças de uma ramificação em outra, normalmente mesclando uma ramificação de recurso à ramificação principal.

---

# Conceitos Fundamentais

## Git

É um sistema distribuído de controle de versão usado para rastrear alterações no conteúdo.

---

## GitLab

É uma plataforma de DevOps completa, fornecida como um único aplicativo.

Ele fornece acesso aos repositórios Git, controlados pelo gerenciamento do código-fonte.

Com GitLab, os colaboradores podem:

- Revisar códigos
    
- Fazer comentários
    
- Trabalhar colaborativamente
    

---

# Ramificação Definitiva

A ramificação definitiva:

- Armazena a versão implantável do seu código.
    
- Deve conter apenas código estável.
    

Quando você quer mudar algo nela:

1. Cria uma nova ramificação.
    
2. Essa nova ramificação é uma cópia exata do código implantável.
    
3. Para criar uma nova ramificação:
    
    - Clique na `main`
        
    - Adicione um nome à nova ramificação
        
    - Selecione **"create branch"**
        

Quando o código original pode ser alterado:

- As ramificações se juntam em uma terceira ramificação entre a original e a ramificação criada.
    

---

# Pull

Pull é uma fusão de ramificações de forma a capturar mudanças.

Ele:

- Disponibiliza alterações confirmadas
    
- Permite que outras pessoas analisem
    
- Permite que outras pessoas usem
    

Um pull pode seguir qualquer confirmação, mesmo que o código esteja inacabado.

---

# Objetivo do Repositório Git

O objetivo é que:

- A ramificação principal seja o único código implantado.
    

Os desenvolvedores podem:

- Fazer alterações na ramificação secundária
    
- Mas somente quando é confirmado com o comando pull que a mudança é realmente colocada em prática
