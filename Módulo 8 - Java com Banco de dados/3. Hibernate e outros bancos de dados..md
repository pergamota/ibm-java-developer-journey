# Hibernate

## O que é Hibernate

O **Hibernate** é um framework **ORM (Object Relational Mapping)** para Java que simplifica operações com banco de dados.

Ele permite trabalhar com **objetos Java em vez de escrever SQL manualmente**.

Antes do Hibernate, com **JDBC**, o desenvolvedor precisava:

- escrever SQL manualmente
    
- abrir conexões
    
- mapear resultados manualmente
    
- lidar com muito código repetitivo
    

O Hibernate automatiza esse processo.

Ele:

- gera SQL automaticamente
    
- gerencia relacionamentos entre tabelas
    
- reduz código manual
    
- facilita manutenção
    

---

## Vantagens do Hibernate

- Independência de banco de dados
    
- Geração automática de SQL
    
- Redução de código repetitivo
    
- Gerenciamento automático de relacionamentos
    
- Controle de transações
    
- Suporte a carregamento **lazy** e **eager**
    

É muito usado em aplicações que possuem:

- grande volume de requisições
    
- muitas entidades relacionadas
    
- necessidade de escalabilidade
    

---

# ORM (Object Relational Mapping)

ORM significa **Mapeamento Objeto-Relacional**.

Ele conecta dois mundos diferentes:

- programação orientada a objetos
    
- bancos de dados relacionais
    

Sem ORM:

Objeto Java → converter manualmente → tabela

Com ORM:

Objeto Java → automaticamente mapeado → tabela

Exemplo de entidade:

```
@Entity  
public class User {  
  
    @Id  
    @GeneratedValue(strategy = GenerationType.IDENTITY)  
    private Long id;  
  
    private String name;  
    private String email;  
  
}
```

Essa classe será mapeada automaticamente para uma tabela no banco.

# Arquitetura do Hibernate

## SessionFactory

Responsável por criar sessões.

Armazena:

- configurações
    
- conexão com banco
    
- mapeamentos de entidades
    

É criado **uma única vez** na aplicação.

---

## Session

Representa uma **conexão com o banco de dados**.

Usada para:

- salvar
    
- atualizar
    
- buscar
    
- deletar objetos
    

Exemplo:

Session session = sessionFactory.openSession();

---

## Transaction

Controla transações no banco.

Garante:

- consistência
    
- integridade
    

Exemplo:

Transaction tx = session.beginTransaction();  
  
session.save(user);  
  
tx.commit();

---

## Query

Permite executar consultas.

Pode usar:

- HQL
    
- SQL
    
- Criteria API
    

---

## Classes de Entidade

São classes Java que representam tabelas.

Cada objeto representa um registro no banco.

---

# Fluxo de Trabalho do Hibernate

1. Carrega configuração do **hibernate.cfg.xml**
    
2. Cria **SessionFactory**
    
3. Abre uma **Session**
    
4. Inicia **Transaction**
    
5. Executa consultas
    
6. Traduz consultas em SQL
    
7. Sincroniza alterações com banco
    
8. Confirma transação
    
9. Fecha sessão

# Anotações Importantes

## @Entity

Define uma classe como entidade.

```
@Entity  
public class Product {

```

---

## @Transient

Campo **não será salvo no banco**.

Usado quando um atributo é apenas para lógica da aplicação.


---

Armazena **dados grandes**.

Exemplo:

- textos grandes

- arquivos
    

```
@Lob
```

---

Usado para mapear datas.

- DATE

Exemplo:

```
@Temporal(TemporalType.DATE)
```

# Estratégias de Carregamento

Hibernate possui duas estratégias.

---

## Lazy Loading (Carregamento Lento)

Entidades relacionadas **não são carregadas imediatamente**.

Elas são carregadas **somente quando acessadas**.

```
@OneToMany(fetch = FetchType.LAZY)  
private List<Order> orders;
```

Vantagem:

- melhor desempenho
    
- menos dados carregados
    

---

## Eager Loading (Carregamento Imediato)

Entidades relacionadas são carregadas **junto com a entidade principal**.

```
@OneToMany(fetch = FetchType.EAGER)  
private List<Order> orders;
```

Usado quando os dados são frequentemente acessados.

---

# HQL (Hibernate Query Language)

HQL é uma linguagem de consulta orientada a objetos.

Ela usa **nomes de entidades e propriedades Java**, não tabelas.

Exemplo HQL:

```
String hql = "FROM Book WHERE author = :author";  
  
Query query = session.createQuery(hql);  
query.setParameter("author", "Robert Martin");  
  
List<Book> books = query.list();
```

---

# SQL Nativo no Hibernate

Também é possível usar SQL diretamente.

```
Query query = session.createNativeQuery(  
"SELECT * FROM books WHERE price > 50"  
);
```

Usado quando:

- migrando sistemas antigos
    
- precisa de otimizações específicas
    

---

# Bancos de Dados NoSQL

**NoSQL** significa **Not Only SQL**.

Eles são usados quando:

- dados são semiestruturados
    
- há necessidade de alta escalabilidade
    
- estrutura muda frequentemente
    
- dados são armazenados em formatos como JSON
    

---

# MongoDB

O **MongoDB** é um banco de dados **NoSQL orientado a documentos**.

Ele armazena dados no formato **JSON/BSON**.

Estrutura:

Database  
  └ Collection  
      └ Document

---

## Coleções

Coleções são semelhantes a tabelas.

Mas **não possuem esquema fixo**.

Cada documento pode ter campos diferentes.

---

## Documento

Exemplo:

```
{  
 "name": "Notebook",  
 "price": 3500,  
 "category": "Eletrônicos"  
}
```

---

# Consultas no MongoDB

### Buscar documentos

db.products.find()

---

### Buscar um documento

db.products.findOne({name: "Notebook"})

---

### Atualizar documento

```
db.products.updateOne(  
 {name:"Notebook"},  
 {$set:{price:3600}}  
)
```

---

### Deletar documento

db.products.deleteOne({name:"Notebook"})

---

# MongoDB no Spring Boot

Para conectar **MongoDB** com **Spring Boot**:

---

## 1 - Configurar conexão

spring.data.mongodb.uri=mongodb://localhost:27017/store

---

## 2 - Criar entidade

```
@Document(collection="products")  
public class Product {  
  
    @Id  
    private String id;  
  
    private String name;  
    private String category;  
  
}
```

---

## 3 - Criar repositório

```
public interface ProductRepository  
extends MongoRepository<Product,String>{  
  
    List<Product> findByCategory(String category);  
  
}
```

Spring cria automaticamente a consulta.

# Bancos de Dados de Séries Temporais

Bancos de séries temporais são otimizados para **dados com tempo associado**.

Exemplo:

- sensores
    
- clima
    
- mercado financeiro
    
- monitoramento de servidores
    

Um banco muito usado é o **InfluxDB**.

---

## Estrutura do InfluxDB

Em vez de tabelas, ele usa:

- **Measurement** → semelhante a tabela
    
- **Tags** → filtros
    
- **Fields** → valores
    

---

## Exemplo de registro

```
sensor_data  
temperature=23.5,location=room1
```

---

# CRUD no InfluxDB

Inserir dado:

```
INSERT sensor_data temperature=25
```

---

Remover dados antigos:

```
DELETE FROM sensor_data WHERE time < now() - 30d
```

Isso remove dados com mais de **30 dias**.

---

# Conectando InfluxDB ao Spring Boot

## 1 - Configurar conexão

```
influx.url=http://localhost:8086  
influx.token=mytoken  
influx.bucket=sensor_data
```

---

## 2 - Definir classe de medição

```
@InfluxMeasurement(name="sensor_data")  
public class SensorData {  
  
    private double temperature;  
  
}
```

---

## 3 - Criar cliente InfluxDB

```
InfluxDBClient client =  
InfluxDBClientFactory.create(url, token);
```

# Anotações importantes do Spring Boot

## @RestController

Define uma classe como controlador REST.

```
@RestController  
public class BookController {  
  
}
```

---

## @Autowired

Injeta dependência automaticamente.

```
@Autowired  
private BookRepository repository;
```

---

## @PostMapping

Cria dados.

```
@PostMapping("/books")  
public Book create(@RequestBody Book book){  
 return repository.save(book);  
}
```

---

## @GetMapping

Recupera dados.

```
@GetMapping("/books")  
public List<Book> getAll(){  
 return repository.findAll();  
}
```

---

## @PutMapping

Atualiza dados.

```
@PutMapping("/books/{id}")  
public Book update(@PathVariable Long id,  
	 @RequestBody Book book){  
	
	 book.setId(id);  
	 return repository.save(book);  
}
```

---

## @DeleteMapping

Remove dados.

```
@DeleteMapping("/books/{id}")  
public void delete(@PathVariable Long id){  
 repository.deleteById(id);  
}
```

# Resumo Final

Hibernate:

- Framework ORM
    
- Converte objetos Java em tabelas
    
- Gera SQL automaticamente
    
- Suporta HQL e SQL nativo
    
- Gerencia relacionamentos e transações
    

NoSQL:

- Alta escalabilidade
    
- Estrutura flexível
    
- Usado para grandes volumes de dados
    

MongoDB:

- Banco orientado a documentos
    
- Usa JSON
    
- Estrutura baseada em coleções e documentos
    

InfluxDB:

- Banco de séries temporais
    
- Otimizado para dados com tempo
    

Spring Boot integra todos esses bancos com facilidade.



# Perguntas e respostas finais:

### 1. O que o comando SELECT INTO faz no contexto do SQL?

- Ele recupera registros do banco de dados.
  
### 2. Descreva uma pool de conexões em JBDC

* Uma coleção de conexões de banco de dados que podem ser reutilizadas.
  
### 3. Qual objeto JDBC é usado especificamente para executar procedimentos armazenados?

* Declaração exigível (_CallableStatement_).
  
### 4. Qual método é usado para executar comandos INSERT, UPDATE e DELETE com um PreparedStatement?

* executeUpdate
  
### 5. Qual é a função do arquivo "schema.sql" em um projeto Spring Boot?

* Para definir o esquema do banco de dados, como tabelas e colunas.
  
### 6. Qual é a finalidade da anotação "@GeneratedValue(strategy = GenerationType.IDENTITY)" em uma entidade JPA?

*  Para especificar que a Chave primária deve ser gerada automaticamente pelo banco de dados.
  
  
### 7. Qual método em uma interface BookRepository seria usado para manter um objeto Book no banco de dados?

* salvar.
  
### 8. Qual é a finalidade do SessionFactory no Hibernate?
  
* Para criar sessões e lidar com as definições de configuração.
  
### 9. No contexto do Hibernate, o que implica a estratégia FetchType.LAZY?

* As entidades relacionadas são buscadas somente quando acessadas.
  
### 10. Em um aplicativo Spring Boot, como a anotação @Param funciona em um método de repositório?

* Mapeia um argumento de método para um parâmetro de consulta dinamicamente.
  
### 11. Qual método é usado para inserir um novo documento em uma coleção do MongoDB? 

* insertOne().
  
### 12. No MongoDB, qual método é usado para atualizar um campo específico em um documento sem afetar outros campos?

* updateOne() com $set.
  
### 13. Em um banco de dados de séries temporais como o InfluxDB, qual é a finalidade de uma medição?

* Para categorizar os pontos de dados, semelhante às tabelas no SQL.
  
### 14. No contexto do InfluxDB, para que são usadas as tags?

* Indexação de pares chave-valor para filtragem e agrupamento de dados.
