# Banco de Dados

Um **banco de dados** é um sistema eletrônico organizado para armazenar, gerenciar e recuperar dados de forma eficiente.

Ele oferece:

- 🔎 Recuperação rápida de dados específicos
    
- 🔐 Integridade dos dados
    
- 💾 Persistência (dados permanecem salvos)
    
- 📈 Escalabilidade
    
- 🔒 Segurança
    
- 👥 Acesso simultâneo (múltiplos usuários ao mesmo tempo)
    

Exemplo: em um site de venda de ingressos, milhares de pessoas podem tentar comprar o mesmo assento simultaneamente — o banco de dados controla isso sem corromper os dados.

---

# Tipos de Banco de Dados

## Banco de Dados Relacional

Armazena dados em **tabelas (linhas e colunas)** com relações entre elas.

Ideal quando:

- os dados possuem estrutura fixa
    
- existem relações claras entre entidades
    

Exemplo de bancos relacionais:

- MySQL
    
- PostgreSQL
    
- Oracle Database
    

---

## Banco de Dados NoSQL

Projetado para **flexibilidade e alta escalabilidade**.

Não utiliza estrutura rígida de tabelas.

Pode armazenar dados como:

- chave–valor
    
- documentos
    
- grafos
    
- colunas
    

Exemplo:

- MongoDB
    

Usado quando:

- dados são não estruturados
    
- estrutura muda com frequência
    
- aplicações de grande escala
    

---

## Banco de Dados de Grafos

Especializado em **relacionamentos complexos**.

Exemplo:

- redes sociais
    
- sistemas de recomendação
    

Armazena conexões entre entidades (nós e arestas).

Exemplo popular:

- Neo4j
    

---

## Banco de Dados de Séries Temporais

Armazena dados com **registro de data e hora**.

Exemplo:

- preços do mercado de ações
    
- sensores IoT
    
- métricas de servidores
    

---

# RDBMS (Sistema de Gerenciamento de Banco de Dados Relacional)

Um **RDBMS** organiza dados em tabelas compostas por:

- Linhas → registros
    
- Colunas → atributos
    

Exemplo tabela Alunos:

|ID|Nome|Idade|
|---|---|---|

Linha → representa um aluno  
Coluna → representa cada campo

---

# Chaves no Banco de Dados

## Chave Primária (Primary Key)

- Identifica exclusivamente cada registro
    
- Não pode ser duplicada
    
- Não pode ser nula
    

Exemplo:

AlunoID

---

## Chave Estrangeira (Foreign Key)

- Cria relação entre tabelas
    
- Referencia a chave primária de outra tabela
    

Exemplo:

Pedido.ClienteID → Cliente.ID

---

# Esquema

Define:

- Tabelas
    
- Colunas
    
- Tipos de dados
    
- Relacionamentos
    
- Restrições (PK, FK, NOT NULL, UNIQUE)
    

---

# Design de Banco de Dados

Começa com o **esquema**.

## Normalização

Processo de dividir dados em tabelas menores e relacionadas para:

- evitar duplicação
    
- melhorar integridade
    
- reduzir inconsistência
    

---

# Índice

Estrutura que acelera a busca de dados.

Exemplo:

CREATE INDEX idx_author ON Books(Author);

Sem índice → busca percorre tabela inteira  
Com índice → busca otimizada

---

# Relacionamentos

## 1️⃣ Um para Um (1:1)

Pessoa ↔ Passaporte

Cada pessoa possui apenas um passaporte.

---

## 2️⃣ Um para Muitos (1:N)

Cliente ↔ Pedidos

Um cliente pode ter vários pedidos.

---

## 3️⃣ Muitos para Muitos (N:N)

Estudantes ↔ Matérias

Requer tabela intermediária:

Student_Course

# SQL (Structured Query Language)

Linguagem padrão para bancos relacionais.

---

## DDL (Data Definition Language)

Define estrutura:

```
CREATE TABLE Books (  
    BookID INT PRIMARY KEY,  
    Title VARCHAR(100),  
    Price DECIMAL(10,2)  
);  
  
ALTER TABLE Books ADD COLUMN Author VARCHAR(100);  
  
DROP TABLE Books;
```

---

## DML (Data Manipulation Language)

Manipula dados:

```
INSERT INTO Books VALUES (1, 'Java', 50.00);  
  
UPDATE Books SET Price = 60 WHERE BookID = 1;  
  
DELETE FROM Books WHERE BookID = 1;  
  
SELECT * FROM Books;
```



---

# Consultas Importantes

## WHERE

SELECT * FROM Books WHERE Author = 'Pergamotinha';

---

## ORDER BY

SELECT * FROM Books ORDER BY Price DESC;

---

## LIMIT

SELECT * FROM Books ORDER BY Price DESC LIMIT 5;

---

## GROUP BY

```
SELECT Author, COUNT(*) AS BookCount  
FROM Books  
GROUP BY Author;
```

---

## HAVING

```
SELECT Author, COUNT(*)   
FROM Books  
GROUP BY Author  
HAVING COUNT(*) > 3;
```

---

## JOIN

```
SELECT Orders.ID, Customers.Name  
FROM Orders  
JOIN Customers ON Orders.CustomerID = Customers.ID;
```

---

## Subconsulta

```
SELECT * FROM Books  
WHERE Price > (SELECT AVG(Price) FROM Books);
```

---

# Procedimento Armazenado

Bloco reutilizável de SQL armazenado no banco.

Exemplo:

```
CREATE PROCEDURE GetExpensiveBooks  
AS  
SELECT * FROM Books WHERE Price > 100;
```

Benefícios:

- Reutilização
    
- Melhor desempenho
    
- Centralização da lógica

# JDBC (Java Database Connectivity)

API Java para conectar aplicações a bancos relacionais.

Principais componentes:

- DriverManager
    
- Connection
    
- Statement
    
- PreparedStatement
    
- ResultSet
    

---

# Arquitetura JDBC

1️⃣ Aplicação Java chama DriverManager  
2️⃣ Driver é selecionado  
3️⃣ Conexão é estabelecida  
4️⃣ Statement executa SQL  
5️⃣ ResultSet armazena resultados

---

# Exemplo de Conexão JDBC

```
Connection connection = DriverManager.getConnection(  
    "jdbc:mysql://localhost:3306/library",  
    "root",  
    "senha"  
);
```

---

# Executando Consulta

```
Statement statement = connection.createStatement();  
  
ResultSet resultSet = statement.executeQuery("SELECT * FROM Books");  
  
while(resultSet.next()) {  
    System.out.println("Book ID: " + resultSet.getInt("BookID"));  
}
```

---

# PreparedStatement (Recomendado)

Evita SQL Injection e melhora desempenho.

```
PreparedStatement ps = connection.prepareStatement(  
    "SELECT * FROM Books WHERE Author = ?");  
  
ps.setString(1, "Pergamotinha");  
  
ResultSet rs = ps.executeQuery();
```

---

# CallableStatement

Executa procedimentos armazenados:

```
CallableStatement cs = 
connection.prepareCall("{call GetExpensiveBooks()}");  
cs.execute();
```


---

# CRUD em JDBC

```
PreparedStatement ps = connection.prepareStatement(

ps.setString(2, "Autor X");  
ps.setDouble(3, 25.99);  
  
ps.executeUpdate();
```

# Connection Pooling

Reutiliza conexões ao invés de abrir/fechar constantemente.

Benefícios:

- Melhor desempenho
    
- Menor sobrecarga
    
- Melhor escalabilidade
    

Exemplo comum em aplicações Spring:

- HikariCP (padrão do Spring Boot)
