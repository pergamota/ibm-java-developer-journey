# Integração Spring Boot com Banco de Dados (JDBC)

Para conectar um aplicativo **Spring Boot** a um banco de dados usando **JDBC**, alguns passos são necessários.

---

## 1️⃣ Adicionar dependências (Maven)

No arquivo **pom.xml**, adicionamos o driver do banco de dados.

Exemplo usando o banco **MySQL**:


```
<dependency>  
    <groupId>mysql</groupId>  
    <artifactId>mysql-connector-j</artifactId>  
</dependency>
```

Se usar Spring Boot:

```
<dependency>  
 <groupId>org.springframework.boot</groupId>  
 <artifactId>spring-boot-starter-jdbc</artifactId>  
</dependency>
```

---

## 2️⃣ Configurar conexão no application.properties

Arquivo:

src/main/resources/application.properties

Exemplo:

```
spring.datasource.url=jdbc:mysql://localhost:3306/bookstore  
spring.datasource.username=root  
spring.datasource.password=password  
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
```

---

## 3️⃣ Configurar logging e segurança

Logs ajudam em:

- depuração
    
- monitoramento
    
- análise de erros
    

Exemplo:

logging.level.org.springframework.jdbc=DEBUG

Importante: evitar registrar **senhas ou dados sensíveis**.

---

# JPA (Java Persistence API)

A **JPA** é uma especificação Java para gerenciar dados relacionais usando **objetos Java**.

Ela permite mapear classes Java para tabelas de banco de dados.

Ou seja:

Classe Java → Tabela SQL  
Objeto → Linha  
Atributo → Coluna

---

## Por que usar JPA?

✔ reduz código SQL repetitivo  
✔ trabalha com objetos Java  
✔ abstrai banco de dados  
✔ facilita CRUD  
✔ melhora produtividade

O Spring Boot normalmente usa a implementação **Hibernate**.

---

# ⚙️ Configuração do JPA no Spring Boot

## 1️⃣ Dependência

```
<dependency>  
 <groupId>org.springframework.boot</groupId>  
 <artifactId>spring-boot-starter-data-jpa</artifactId>  
</dependency>
```

---

## 2️⃣ Configuração no application.properties

```
spring.datasource.url=jdbc:mysql://localhost:3306/bookstore  
spring.datasource.username=root  
spring.datasource.password=password  
  
spring.jpa.hibernate.ddl-auto=update  
spring.jpa.show-sql=true
```

### Explicação

`ddl-auto` pode ser:

|valor|função|
|---|---|
|create|recria banco|
|update|atualiza tabelas|
|validate|valida estrutura|
|none|não altera|

# Entidades (Entities)

Uma entidade representa uma tabela no banco.

Exemplo:

```
@Entity  
public class Book {  
  
 @Id  
 @GeneratedValue(strategy = GenerationType.IDENTITY)  
 private Long id;  
  
 private String title;  
 private String author;  
 private double price;  
  
}
```

Principais anotações:

|anotação|função|
|---|---|
|@Entity|define entidade|
|@Table|define tabela|
|@Id|chave primária|
|@GeneratedValue|geração automática|
|@Column|define coluna|

# Repositórios (Spring Data JPA)

Spring Boot simplifica acesso ao banco usando **repositórios**.

Exemplo:

```
public interface BookRepository extends JpaRepository<Book, Long> {  
}
```

Isso já fornece automaticamente:

- save()
    
- findAll()
    
- findById()
    
- delete()
    

---

### Exemplo de método personalizado

List<Book> findByAuthor(String author);

Spring gera a consulta automaticamente.

--- 

# Operações CRUD com JPA

## Criar

bookRepository.save(new Book("Java", "Autor X", 40));

---

## Ler

List<Book> books = bookRepository.findAll();

---

## Atualizar

```
Book book = bookRepository.findById(1L).get();  
book.setPrice(50);  
bookRepository.save(book);
```
---

## Deletar

bookRepository.deleteById(1L);

# Validação de Dados

Validação garante que os dados estejam corretos antes de salvar.

Spring usa **Bean Validation (JSR-380)**.

---

## Anotações comuns

|anotação|função|
|---|---|
|@NotNull|campo obrigatório|
|@Size|tamanho mínimo/máximo|
|@Pattern|valida formato|
|@Email|valida email|
|@Min|valor mínimo|
|@Max|valor máximo|

---

### Exemplo

```
public class User {  
  
 @NotNull  
 private String name;  
  
 @Email  
 private String email;  
  
 @Size(min=6)  
 private String password;  
  
}
```

---

# @Valid

Valida automaticamente dados recebidos em APIs.

Exemplo:

```
@PostMapping  
public ResponseEntity<User> createUser(@Valid @RequestBody User user){  
 return ResponseEntity.ok(user);  
}
```

---

# Tratamento de Erros

## @ControllerAdvice

Centraliza tratamento de exceções.

```
@ControllerAdvice  
public class GlobalExceptionHandler {  
  
 @ExceptionHandler(Exception.class)  
 public ResponseEntity<String> handleError(Exception ex){  
  return ResponseEntity.badRequest().body(ex.getMessage());  
 }  
  
}
```

# API RESTful

Uma **API RESTful** permite comunicação entre cliente e servidor usando HTTP.

Métodos principais:

|método|função|
|---|---|
|GET|buscar dados|
|POST|criar|
|PUT|atualizar|
|DELETE|excluir|

---

## Exemplo Controller

```
@RestController  
@RequestMapping("/books")  
public class BookController {  
  
 @Autowired  
 private BookRepository repository;  
  
 @GetMapping  
 public List<Book> getBooks(){  
  return repository.findAll();  
 }  
  
}
```

# Transações

Transações garantem integridade dos dados.

Usamos:

@Transactional

Exemplo:

```
@Transactional  
public void transferMoney(Account a, Account b){  
}
```

Se algo falhar → rollback automático.
