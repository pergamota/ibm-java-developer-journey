
# Exceções em Java

## O que são Exceções

Exceções são eventos que **interrompem o fluxo normal do programa** durante a execução.  
Elas ocorrem em situações como:

- Divisão de um número por zero
    
- Acesso a um índice inexistente
    
- Tentativa de acessar um arquivo que não existe


## Erros em Java

Erros estão relacionados a **problemas graves**, que geralmente **não podem ser resolvidos pelo programador**, como:

- Falta de memória da JVM
    
- Vazamento de memória
    
- Estouro de pilha (StackOverflowError)


## Diferença entre Erros e Exceções

- **Erro**: problema crítico que uma aplicação razoável **não deve tentar tratar**.
    
- **Exceção**: situação que uma aplicação **pode prever, detectar e tratar**.


# Tipos de Exceções em Java

## Exceções Verificadas (Checked Exceptions)

São exceções **verificadas em tempo de compilação**.  
O compilador exige que elas sejam tratadas ou declaradas.

### Características

- Devem ser tratadas com `try-catch` **ou**
    
- Declaradas na assinatura do método com `throws`
    

### Onde são usadas

- Operações com arquivos
    
- Conexões com banco de dados
    
- Comunicação em rede


## Exceções Não Verificadas (Unchecked Exceptions)

São exceções que **ocorrem em tempo de execução**.  
O compilador **não exige tratamento explícito**.

### Características

- Geralmente indicam **erros de programação**
    
- Podem ser evitadas com código correto
    
- Não precisam ser declaradas ou capturadas obrigatoriamente


## Exceções Personalizadas

Ocorrem quando você **cria suas próprias classes de exceção** em Java, estendendo `Exception` ou `RuntimeException`.


# Tratamento de Exceções

## Bloco try-catch

Usado para capturar e tratar exceções durante a execução do programa.

### Sintaxe

try {
    // Código que pode gerar uma exceção
} catch (Exception e) {
    // Manipula a exceção
}


### Benefícios

- Evita a falha completa do programa
    
- Permite respostas apropriadas a erros
    
- Facilita a detecção de problemas em tempo de execução


## Múltiplos Blocos try-catch

Permitem tratar **diferentes exceções separadamente**.

### Benefícios

- Tratamento mais claro e específico
    
- Código mais organizado
    
- Respostas adequadas para cada tipo de erro


## Bloco finally

O bloco `finally` é executado **independentemente de ocorrer ou não uma exceção**.

### Uso Correto

- Fechar arquivos
    
- Encerrar conexões com banco de dados
    
- Liberar recursos do sistema
    

### Exemplo de Uso

Mesmo que uma exceção ocorra no `try` ou seja capturada no `catch`, o `finally` será executado.

### Uso Incorreto

- Colocar `return` dentro do `finally`
    
- Não tratar exceções geradas dentro do próprio bloco `finally`
    

### Observação Importante

O bloco `finally` **sempre será executado**, exceto se:

- O programa falhar
    
- O sistema for encerrado à força


# Palavra-chave throws

## O que é throws

A palavra-chave `throws` é usada na **assinatura de um método**, indicando que ele pode lançar uma ou mais exceções.

### Benefícios

- Separação de responsabilidades
    
- Código mais limpo e organizado
    
- Clareza sobre quais erros podem ocorrer
    
- Melhores práticas de design e tratamento de erros


# Exceções Verificadas Comuns

## IOException

Usada em operações de:

- Leitura e gravação de arquivos
    
- Comunicação de rede


## FileNotFoundException

Ocorre quando um arquivo não é encontrado.


## ClassNotFoundException

Ocorre ao tentar carregar classes dinamicamente em tempo de execução.


## Benefícios das Exceções Verificadas

- **Segurança em tempo de compilação**
    
- **Erros recuperáveis**, como entrada inválida ou arquivo inexistente
    
- **Contratos de API claros**, informando possíveis falhas
    
- Incentiva **programação defensiva**


# Exceções de Tempo de Execução (Runtime Exceptions)

## NullPointerException

Ocorre ao tentar acessar um objeto com referência nula.


## ArithmeticException

Ocorre em operações matemáticas inválidas, como divisão por zero.


## ArrayIndexOutOfBoundsException

Ocorre ao acessar um índice inexistente em um array.


## Benefícios das Exceções de Tempo de Execução

- **Simplicidade**, pois não exigem tratamento obrigatório
    
- Indicam **falhas de lógica no código**
    
- Ajudam no desenvolvimento e depuração
    
- Menor impacto no controle de fluxo do programa
