
## Conceitos de Desenvolvimento Web

### Front-end

Desenvolvedores front-end trabalham na parte do site que o usuário visualiza e interage diretamente. Isso inclui botões, formulários, layouts, textos, imagens e animações.

#### Estrutura física (Front-end)

- Textos
    
- Links
    
- Imagens
    
- Vídeos
    
- Botões


### HTML (HyperText Markup Language)

Usado para criar a **estrutura básica de um site**, organizando o conteúdo.

Exemplos de elementos criados com HTML:

- Parágrafos
    
- Links
    
- Títulos
    
- Imagens
    
- Vídeos
    
- Botões


### CSS (Cascading Style Sheets)

Fornece um método padrão para **definir e gerenciar o estilo visual** de um site.

Funções do CSS:

- Definir cores, fontes e espaçamentos
    
- Deixar o site visualmente bonito e uniforme
    
- Garantir compatibilidade com diferentes navegadores
    
- Adaptar o site para PC, celular e tablet


### JavaScript

Usado em conjunto com HTML e CSS para **adicionar interatividade**.

Exemplo:

- HTML cria um botão
    
- CSS define a cor e o estilo
    
- JavaScript define o comportamento do botão


### Design Adaptativo e Responsivo

Permite que o site se **ajuste automaticamente ao tamanho da tela** do dispositivo utilizado, como:

- Computador
    
- Celular
    
- Tablet


## Bibliotecas e Frameworks Front-end

### React.js

- Biblioteca **JavaScript**
    
- Focada na **interface do usuário (UI)**
    
- Cria componentes reutilizáveis
    
- Renderiza páginas de forma eficiente
    
- Muito usada em aplicações modernas e interativas


### Vue.js

- Atua na camada de interface do usuário
    
- Trabalha com botões e componentes visuais
    
- Flexível
    
- Integra bem com outras ferramentas


## Back-end

O back-end é responsável por **processar as solicitações feitas pelo usuário**.

Funções do desenvolvedor back-end:

- Processar requisições
    
- Fornecer dados
    
- Garantir segurança
    
- Gerenciar regras de negócio
    

O usuário interage com o front-end, que envia solicitações ao back-end, e o back-end responde com dados ou ações.


### SQL

Linguagem usada para **interagir com bancos de dados**.

Permite:

- Guardar dados
    
- Buscar dados
    
- Alterar dados
    
- Apagar dados


### API

Código que permite a **comunicação entre sistemas**.

Características:

- Recupera dados do back-end
    
- Geralmente funciona em ambiente de nuvem
    
- Faz a ponte entre front-end e back-end


### Routing

Define os **caminhos (rotas)** de um site ou aplicação.


### Endpoint

Pode ser:

- Uma API
    
- Um caminho específico (rota)
    

É o ponto onde uma requisição chega.


## Programação em Pares (Pair Programming)

Dois desenvolvedores trabalham juntos no mesmo computador, compartilhando conhecimento e melhorando a comunicação.

### Estilos de programação em pares

#### Driver / Navigator

- Driver: escreve o código
    
- Navigator: revisa e orienta
    
- Os papéis são alternados
    

#### Ping-Pong

- Baseado em testes
    
- Um escreve o teste
    
- O outro escreve o código para passar no teste
    

#### Strong Style

- Toda ideia deve ser explicada ao parceiro antes de virar código
    
- Evita interrupções no fluxo de pensamento
    

### Pontos negativos

- Pode ser cansativo
    
- Exige mais foco
    
- Compromissos pessoais podem atrapalhar
    
- Uma pessoa pode acabar controlando tudo


## Ferramentas para Desenvolvimento em Nuvem

### Sistema de Controle de Versão

Monitora:

- Alterações feitas no código
    
- Quando foram feitas
    
- Por quem
    

Permite recuperar versões anteriores caso algo dê errado.


### Git e GitHub

- Git: controla versões do código
    
- GitHub: armazena e gerencia repositórios
    

Permite:

- Rastrear alterações
    
- Trabalhar com ramificações (branches)
    
- Desenvolvimento colaborativo


### Libraries

Coleções de código prontas que facilitam o desenvolvimento.

Exemplo:

- `math.h` em C para operações matemáticas
    

### Frameworks

Fornecem uma **estrutura padrão** para criar aplicações.

Exemplo:

- AngularJS: framework baseado em JavaScript para aplicações web dinâmicas
    

Frameworks mais controladores são chamados de **frameworks opinativos**.


## CI/CD

Práticas de:

- **Integração Contínua (CI)**
    
- **Entrega ou Implantação Contínua (CD)**
    

Funcionamento:

- CI compila e testa o código
    
- CD implanta as alterações automaticamente


### Build Tools

Ferramentas que:

- Automatizam tarefas repetitivas
    
- Compilam o código
    
- Transformam o código em binário


### Package

Inclui compiladores e ambientes como:

- Linux
    
- Windows
    
- iOS
 

## Software Stacks

Combinação de tecnologias usadas para criar aplicações.

Inclui:

- Front-end
    
- Back-end
    
- Frameworks
    
- Segurança
    
- Rede
    

Quanto mais complexo o sistema, mais camadas a pilha terá.


### Exemplos de Stacks

#### Python + Django

- Código aberto
    
- Ideal para aplicações grandes
    
- Permite rápidas mudanças
    

#### Ruby on Rails

- Usado para transferência de dados
    
- Auxilia no desenvolvimento do front-end


### Comparação entre MEAN, MEVN e LAMP

#### MEAN

- Usa apenas JavaScript
    
- Código aberto
    
- Documentação reutilizável
    
- Não indicada para aplicações de larga escala
    

#### MEVN

- Similar à MEAN
    
- Tecnologia mais nova
    
- Menos bibliotecas disponíveis
    

#### LAMP

- Código aberto
    
- Mais antiga
    
- Fácil de encontrar suporte
    
- Menos flexível por ser baseada em Linux
