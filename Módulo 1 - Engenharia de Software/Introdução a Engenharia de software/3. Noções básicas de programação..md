
## Linguagens de Programação

Linguagens de programação nos ajudam a dizer ao computador o que ele precisa fazer.


## Linguagens Interpretadas (Script)

Linguagens interpretadas, como **Python** e **JavaScript**, executam o código por meio de um **intérprete**, que traduz o código-fonte para linguagem de máquina no momento da execução.

Características:

- O código é lido linha por linha
    
- Precisa ser interpretado toda vez que é executado
    
- Normalmente usadas para scripts e automação
    
- Funcionam em diferentes sistemas operacionais, desde que exista um intérprete adequado
    

### Exemplos

- **JavaScript**: linguagem de script simples cujo interpretador é o navegador web
    
- **Python**: muito usada para automação, ciência de dados e back-end
    
- **Lua**: usada em jogos leves e aplicações de uso geral


## Linguagens Compiladas

Linguagens compiladas, como **C** e **C++**, geram um **arquivo executável** a partir do código-fonte.

Funcionamento:

- Um compilador converte todo o código em linguagem de máquina
    
- O resultado é um arquivo executável
    
- O programa pode ser executado diretamente no sistema
    

Características:

- Normalmente usadas para softwares mais complexos
    
- Exigem mais tempo para escrever e compilar
    
- Executam mais rápido, pois já estão convertidas para linguagem do computador
    

Exemplos de programas compilados:

- Sistemas operacionais
    
- Aplicativos como Spotify e Windows


## Script

Scripts servem para **automatizar tarefas**. Em vez de executar ações manualmente no computador, o desenvolvedor escreve instruções que o sistema interpreta e executa.


## Diferença entre Linguagens Interpretadas e Compiladas

### Linguagens Interpretadas

- Código executado linha por linha
    
- Mais fáceis de aprender
    
- Muito usadas em sites e automações
    
- Portáveis entre sistemas operacionais
    

### Linguagens Compiladas

- Código transformado em arquivo executável
    
- Mais difíceis de aprender
    
- Melhor desempenho
    
- Indicadas para sistemas maiores e mais complexos    


## Exemplos de Linguagens

- **C, C++ e C#**: linguagens compiladas
    
- **Java**: compilada para bytecode e executada na JVM, podendo rodar em Windows, macOS e Linux
    
- **Python e JavaScript**: linguagens interpretadas, usadas em múltiplos sistemas operacionais


## Nível das Linguagens de Programação

### High-level (Alto nível)

- Mais próximas da linguagem humana
    
- Mais fáceis de entender e depurar
    
- Exemplos: Python, SQL
    

### Low-level (Baixo nível)

- Mais próximas da linguagem de máquina
    
- Usam símbolos e instruções específicas do hardware
    
- Exemplos: Assembly (ARM, MIPS)


## Linguagens de Consulta (Query Languages)

Linguagens de consulta são usadas para **solicitar informações de bancos de dados**.

Características:

- Banco de dados e aplicação precisam “falar a mesma língua”
    
- Usam comandos predefinidos para realizar consultas
    

### Exemplo mais comum

- **SQL**
    

Outros exemplos:

- AQL
    
- CQL
    
- Datalog
    
- DMW
    

### SQL x NoSQL

- **SQL**: usa esquemas predefinidos
    
- **NoSQL**: usa esquemas dinâmicos
    

### Principais comandos SQL

- CREATE
    
- INSERT
    
- UPDATE
    

Esses comandos são chamados de **declarações**.


## Linguagem de Montagem (Assembly)

- Linguagem de baixo nível
    
- Mais próxima do hardware
    
- Usa um **montador (assembler)**, não um compilador ou intérprete
    
- Permite comunicação direta com o processador
    
- Indica o que o processador deve fazer e onde encontrar os dados


## Organização do Código

A organização do código é essencial para:

- Melhor leitura
    
- Manutenção
    
- Design de software
    

### Principais métodos de organização

### Pseudocódigo

Descrição básica e de alto nível de um algoritmo.

Características:

- Não segue uma linguagem específica
    
- Foca na lógica do problema
    
- Facilita o planejamento do projeto
    
- Ajuda no compartilhamento de ideias
    

Benefícios:

- Simples
    
- Ajuda a pensar na solução antes do código
    
- Menos detalhado que fluxogramas


### Fluxogramas

Representação visual das etapas de um algoritmo.

Características:

- Uso de setas e símbolos
    
- Indica a ordem das operações
    
- Muito usado para documentação e ensino
    

Elementos comuns:

- Início/Fim
    
- Processo
    
- Decisão
    
- Dados
    
- Conectores
    

Mais indicado para problemas menores.


## Lógica de Programação

Computadores utilizam **lógica booleana**, baseada em dois valores:

- Verdadeiro
    
- Falso
    

Existem dois tipos principais de lógica:


### Lógica de Ramificação

O programa toma decisões com base em condições.

Características:

- Cada condição pode gerar um novo caminho
    
- Não há limite para o número de ramificações
    

Estruturas comuns:

- IF
    
- ELSE
    
- SWITCH


### Lógica de Loop

Executa um conjunto de instruções repetidamente até que uma condição seja atendida.

Estruturas comuns:

- WHILE
    
- FOR


## Containers

Containers incluem estruturas como:

- Vetores
    
- Matrizes
    

São usados para armazenar e organizar dados.


## Conceito de Objetos (Programação Orientada a Objetos – POO)

POO é uma metodologia focada em **objetos**, e não apenas em funções.

Características:

- Dados e comportamentos são agrupados
    
- Diferente da programação estruturada, que separa dados e funções
    

Elementos principais:

- **Propriedades (estado)**
    
- **Métodos (comportamentos)**
    

Objetos de software podem representar:

- Janelas
    
- Contas de usuário
    
- Componentes de sistemas
    

Os objetos:

- Armazenam dados em variáveis
    
- Expõem comportamentos por meio de métodos
