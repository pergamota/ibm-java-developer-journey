
# Arquitetura de Software

## Conceito de Arquitetura de Software

Arquitetura de software é a organização do sistema e serve como um modelo que os desenvolvedores utilizam para criar e integrar os componentes do software. Ela define como os componentes interagem entre si, o ambiente operacional e os princípios usados no projeto.

Além disso, a arquitetura aborda requisitos não funcionais, como desempenho, escalabilidade, manutenção e confiabilidade. Ela equilibra as necessidades do projeto, facilita a comunicação entre desenvolvedores e demais membros da equipe, aumenta a vida útil do sistema e auxilia na definição da pilha tecnológica, composta por linguagens, bibliotecas e frameworks.


## Artefatos de Arquitetura

Artefatos são documentos e modelos utilizados para representar e comunicar a arquitetura do sistema.


### SDD (Software Design Document)

O SDD é uma coleção de especificações técnicas que descrevem como o design deve ser implementado, incluindo restrições, requisitos e descrições funcionais.


### Diagramas Arquitetônicos

Diagramas arquitetônicos exibem os componentes do sistema e como eles interagem entre si.


## Software Design

Software design é o processo no qual os atributos estruturais e comportamentais do sistema são documentados antes do desenvolvimento. Esse processo envolve a modelagem do software por meio de representações visuais, como diagramas, frequentemente utilizando UML.


## Coesão e Acoplamento

### Coesão

Refere-se ao grau em que elementos funcionalmente relacionados estão agrupados dentro de um módulo.

### Acoplamento

Representa o nível de dependência e comunicação entre diferentes módulos do sistema.

## Módulos

Módulos são as principais unidades funcionais do sistema e geralmente aparecem como blocos em diagramas UML. Submódulos são componentes menores conectados a esses módulos principais.


## UML (Unified Modeling Language)

UML é uma linguagem padronizada de modelagem utilizada no desenvolvimento de software. Seus diagramas podem ser classificados em estruturais ou comportamentais e podem ser aplicados a sistemas desenvolvidos em diferentes linguagens de programação.

### Diagramas de Interação

Diagramas de interação modelam o comportamento dinâmico de um sistema, ajudando a visualizar objetos e seus relacionamentos ao longo do tempo.


## OOAD (Object-Oriented Analysis and Design)

OOAD é uma abordagem de análise e design voltada para sistemas desenvolvidos com Programação Orientada a Objetos.

### Classes em POO

Classes funcionam como modelos para a criação de objetos. Um objeto é uma instância de uma classe e possui atributos e métodos.

### Diagrama de Classes

O diagrama de classes, baseado em UML, representa a estrutura do sistema e mostra como as classes se relacionam entre si.


## Componentes

Componentes são partes de um aplicativo que se integram a outras partes do sistema. Eles podem ser reutilizáveis, substituíveis, independentes, extensíveis, encapsulados e não específicos.

Uma API pode ser considerada um componente quando pode ser reutilizada em diferentes sistemas.


## Arquitetura Baseada em Componentes

Esse tipo de arquitetura opera em um nível mais alto de abstração do que a POO, definindo componentes independentes e fracamente acoplados que trabalham juntos para formar o aplicativo.


## Serviços

Serviços são semelhantes a componentes, porém projetados para implantação independente e reutilização. Eles atendem a necessidades específicas do negócio, como verificação de crédito ou cálculo de pagamentos.


## Arquitetura Orientada a Serviços (SOA)

SOA é baseada em serviços fracamente acoplados que se comunicam por meio de protocolos de rede, permitindo a criação de sistemas distribuídos.


## Sistemas Distribuídos

Sistemas distribuídos são compostos por vários serviços localizados em máquinas diferentes que se comunicam por troca de mensagens, geralmente via HTTP. Eles continuam operando mesmo que algum subsistema falhe.


## Tipos de Arquitetura

### Arquitetura em Duas Camadas

O servidor hospeda e gerencia os serviços, enquanto o cliente fornece a interface e faz solicitações. Um exemplo é o WhatsApp.

### Arquitetura em Três Camadas

Divide o sistema em camadas independentes que interagem apenas com camadas adjacentes. Um exemplo é o Instagram.

### Arquitetura Orientada a Eventos (Event-Driven)

Baseada na produção e consumo de eventos. Produtores geram eventos e consumidores os processam. Um exemplo é o Uber.

### Microservices

Divide a aplicação em serviços independentes que se comunicam por meio de APIs.

### Peer-to-Peer

Os participantes atuam simultaneamente como clientes e servidores, compartilhando recursos diretamente, como ocorre em torrents e criptomoedas.


## Ambientes de Aplicação

### Pré-produção

Inclui ambientes de desenvolvimento, controle de qualidade e preparação.

### Ambiente de Preparação

Replica o ambiente de produção, mas não é acessível aos usuários finais.

### Produção

Contém toda a pilha de hardware e software necessária para executar o aplicativo para os usuários finais.


## Modelos de Implantação

### Implantação Local

O sistema fica dentro da organização, protegido por firewall, oferecendo maior controle e segurança.

### Nuvem Pública

Utiliza infraestrutura compartilhada do provedor, oferecendo escalabilidade e menor custo.

### Nuvem Privada

Infraestrutura dedicada a uma única organização, oferecendo maior segurança.

### Nuvem Híbrida

Combina nuvem pública e privada, equilibrando custo, segurança e flexibilidade.


## Infraestrutura de Suporte

### Firewall

Monitora e controla o tráfego de rede, protegendo a rede interna.

### Balanceadores de Carga

Distribuem o tráfego entre servidores para evitar sobrecarga.

### Servidores Web

Fornecem páginas, arquivos e recursos para clientes.

### Servidores de Aplicação

Executam a lógica de negócios do sistema.

### Proxy Server

Atua como intermediário, oferecendo segurança, cache e otimização.

### Banco de Dados

Coleção de dados gerenciada por um DBMS, responsável pelo armazenamento e controle das informações.
