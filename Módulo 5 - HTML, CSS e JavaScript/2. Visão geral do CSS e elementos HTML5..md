
# HTML5, CSS e Responsividade — Resumo Organizado

## HTML5 — Elementos Estruturais

O HTML5 introduz uma estrutura baseada em **divisões semânticas**, facilitando a organização do layout das páginas, como cabeçalho, navegação, conteúdo principal e rodapé.

### Uso da tag `<div>`

A tag `<div>` define uma divisão genérica da página e é usada para **agrupar conteúdo**.

- Pode conter qualquer tipo de conteúdo
    
- Não possui significado semântico
    
- Muito usada para layout e estilização
    
- Útil quando vários elementos compartilham o mesmo estilo
    

Exemplo conceitual:

- `<p>` → parágrafo (conteúdo textual)
    
- `<div>` → agrupamento genérico de conteúdo
    

---

## Elementos Semânticos do HTML5

Em vez de usar apenas `<div>` ou `<span>`, o HTML5 oferece **elementos semânticos**, que descrevem melhor o papel do conteúdo.

### Principais elementos semânticos

- `<article>`  
    Define um conteúdo independente, como notícias, posts de blog ou comentários.
    
- `<section>`  
    Define uma seção lógica do documento, como capítulos ou partes temáticas.
    
- `<header>`  
    Agrupa títulos, logotipos e elementos introdutórios da página ou seção.
    
- `<footer>`  
    Define o rodapé da página ou de uma seção.
    
- `<aside>`  
    Conteúdo complementar ao principal (links extras, propagandas, notas).
    
- `<nav>`  
    Agrupa links de navegação do site.
    
- `<figure>`  
    Usado para imagens, gráficos, diagramas.
    
- `<figcaption>`  
    Define a legenda de um `<figure>`.
    

---

## Elementos de Entrada (Inputs HTML5)

HTML5 trouxe novos tipos de entrada para formulários, melhorando validação e experiência do usuário.

### Tipos de input

- `type="date"`  
    Entrada de data (ano, mês e dia), sem fuso horário.
    
- `type="number"`  
    Entrada numérica, com valores mínimos e máximos.
    
- `type="range"`  
    Intervalo numérico (ex: controle deslizante).
    
- `type="tel"`  
    Entrada de número de telefone.
    
- `type="url"`  
    Valida se o valor inserido é uma URL válida.
    
- `list="some_list"`  
    Permite preenchimento automático com sugestões.
    

### Atributos importantes

- `placeholder`  
    Exibe uma dica de formato dentro do campo.
    
- `required`  
    Indica que o campo é obrigatório.
    

### Validação

- Validação nativa do HTML5 pode não ser compatível com navegadores antigos
    
- Alternativa: usar **JavaScript** ou **jQuery**
    

---

## Acessibilidade e Renderização

- Documentos HTML podem ser renderizados:
    
    - Em navegadores
        
    - Por leitores de tela (acessibilidade)
        
- Importante separar **conteúdo (HTML)** de **design (CSS)**

# CSS — Cascading Style Sheets

CSS define **como os elementos HTML são exibidos**.

## O que o CSS controla

- Fontes
    
- Cores
    
- Fundos
    
- Bordas
    
- Tamanhos
    
- Espaçamentos
    
- Layout
    

### Conceito de Cascata

- Estilos podem ser herdados de elementos pais
    
- Algumas propriedades não são herdadas automaticamente
    

---

## Exemplo de CSS Básico

```
body {
    color: #000000;
    margin: 0;
    padding: 0;
}
```

---

## Tipos de Layout

### Layout Fluido

- Dimensões flexíveis
    
- Se adapta ao tamanho da tela
    
- Usa porcentagem (`%`)
    

### Layout Fixo

- Dimensões fixas
    
- Não se adapta ao tamanho da tela
    
- Usa pixels (`px`)
    

---

## Formas de Aplicar CSS

### CSS Inline (Embutido)

- Aplicado diretamente no elemento
    
- Usa o atributo `style`
    

**Prós:**

- Simples
    
- Rápido para ajustes pequenos
    

**Contras:**

- Difícil de manter
    
- Viola separação de conteúdo e estilo
    

---

### CSS Interno

- Definido dentro da tag `<style>`
    
- Afeta apenas uma página
    

**Prós:**

- Centraliza estilos da página
    

**Contras:**

- Não reutilizável
    
- Aumenta o tamanho da página
    

---

### CSS Externo (Recomendado)

- Arquivo `.css` separado
    
- Vinculado ao HTML com `<link>`
    

**Prós:**

- Código limpo
    
- Reutilização
    
- Melhor manutenção
    

**Contras:**

- Requer requisição HTTP adicional
    

### Ordem de Prioridade

1. CSS Inline
    
2. CSS Interno
    
3. CSS Externo
    

---

## Seletores CSS

Seletores definem **quais elementos receberão o estilo**.

### Seletor de tag

- Aplica estilo a todas as ocorrências de uma tag
    

Exemplo:

```
p {
    color: blue;
}
```

# Frameworks CSS

## Tipos de Estruturas CSS

### Vanilla CSS

- CSS puro
    
- Total controle
    
- Mais trabalho manual
    

---

### Utility-first

- Classes pequenas e reutilizáveis
    
- Estilo direto no HTML
    
- Exemplo: Tailwind CSS
    

---

### Componentes

- Componentes prontos
    
- Menos flexibilidade
    
- Exemplo: Bootstrap
    

---

## Tailwind CSS

- Abordagem **utility-first**
    
- Classes combináveis
    
- Alta personalização
    
- Menos código CSS customizado
    

---

## Bootstrap

- Componentes prontos
    
- Sistema de grid de 12 colunas
    
- Mais fácil para iniciantes
    
- Menos flexível
    

---

## Breakpoints

Breakpoints permitem que o layout se adapte a diferentes tamanhos de tela:

- Celulares
    
- Tablets
    
- Desktops

# Responsividade com Tailwind CSS

```
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Responsive Layout - Tailwind</title>
    <link href="https://cdn.jsdelivr.net/npm/tailwindcss@2.2.19/dist/tailwind.min.css" rel="stylesheet">
</head>
<body class="bg-gray-100">

<header class="bg-red-500 text-white text-center p-4">Header</header>
<nav class="bg-red-500 text-white text-center p-4">Navigation</nav>

<div class="flex flex-col md:flex-row">
    <aside class="bg-yellow-300 p-4 w-full md:w-1/4">Aside</aside>
    <section class="bg-white p-4 flex-grow">Main Section</section>
    <article class="bg-green-300 p-4 w-full md:w-1/4">Article</article>
</div>

<footer class="bg-blue-500 text-white text-center p-4">Footer</footer>

</body>
</html>
```

### Explicação das classes

- `flex`, `flex-col` → layout flexível em coluna
    
- `md:flex-row` → muda para linha em telas médias
    
- `w-full`, `md:w-1/4` → largura responsiva
    
- `p-4` → espaçamento interno
    
- `bg-*` → cores de fundo

# Responsividade com Bootstrap

```
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Responsive Layout - Bootstrap</title>
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0-alpha1/dist/css/bootstrap.min.css" rel="stylesheet">
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0-alpha1/dist/js/bootstrap.bundle.min.js"></script>
</head>
<body>

<header class="bg-danger text-white text-center p-3">Header</header>

<nav class="navbar navbar-expand-lg bg-light">
    <div class="container-fluid">
        <a class="navbar-brand" href="#">Brand</a>
        <button class="navbar-toggler" type="button" data-bs-toggle="collapse" data-bs-target="#navbarNav">
            <span class="navbar-toggler-icon"></span>
        </button>
        <div class="collapse navbar-collapse" id="navbarNav">
            <ul class="navbar-nav">
                <li class="nav-item"><a class="nav-link" href="#">Home</a></li>
                <li class="nav-item"><a class="nav-link" href="#">About</a></li>
                <li class="nav-item"><a class="nav-link" href="#">Contact</a></li>
            </ul>
        </div>
    </div>
</nav>

<div class="container my-3">
    <div class="row">
        <aside class="col-12 col-md-3 bg-warning p-3">Aside</aside>
        <section class="col-12 col-md-6 bg-white p-3">Main Section</section>
        <article class="col-12 col-md-3 bg-success p-3">Article</article>
    </div>
    <footer class="bg-primary text-white text-center p-3 mt-3">Footer</footer>
</div>

</body>
</html>
```

## Comparação: Tailwind vs Bootstrap

### Tailwind CSS

- Utility-first
    
- Alta personalização
    
- Menor tamanho final
    
- Exige conhecimento de CSS
    

### Bootstrap

- Componentes prontos
    
- Mais fácil para iniciantes
    
- Menos flexível
    
- Arquivo maior
