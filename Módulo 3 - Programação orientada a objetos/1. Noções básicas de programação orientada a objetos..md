
# Programação Orientada a Objetos (POO)

## Propriedades (Atributos)

Propriedades são as **características de um objeto**.

Exemplo:  
Se o objeto for um **carro**, suas propriedades podem ser:

- Cor
    
- Velocidade
    
- Modelo
    

---

## Métodos

Métodos representam as **ações que um objeto pode executar**.

Exemplo:  
No caso de um carro, os métodos podem ser:

- Dirigir
    
- Buzinar
    
- Estacionar
    

Eles definem o **comportamento do objeto**.

---

# Pilares da Programação Orientada a Objetos

## Herança

Herança permite criar **novas classes com base em classes existentes**.

Ela funciona como uma **árvore genealógica**, onde:

- Uma classe principal (superclasse) passa atributos e métodos
    
- Classes filhas (subclasses) herdam esses recursos
    

### Exemplo

A classe `Carro` pode gerar subclasses como:

- CarroElétrico
    
- CarroEsportivo
    

Essas subclasses:

- Herdam propriedades e métodos da classe `Carro`
    
- Podem adicionar ou modificar comportamentos próprios
    

---

## Encapsulamento

Encapsulamento é o conceito de **agrupar dados e métodos em uma única unidade**, geralmente uma classe.

### Objetivo

- Esconder os detalhes internos do objeto
    
- Expor apenas o que é necessário para o uso externo
    

### Exemplo

Ao usar uma **TV**, você:

- Troca o canal
    
- Ajusta o volume
    

Mas **não sabe como isso funciona internamente**.  
Isso é encapsulamento.

### Benefícios

- Protege os dados
    
- Controla o acesso aos atributos
    
- Permite que os dados sejam acessados apenas por métodos específicos
    

---

## Polimorfismo

Polimorfismo permite que **objetos diferentes sejam tratados como instâncias da mesma classe base**.

### Características

- Um mesmo método pode ter comportamentos diferentes
    
- O comportamento depende do tipo real do objeto
    

### Exemplo

Um **helicóptero** e um **foguete** podem ter o método `voar`:

- Helicóptero voa com hélices
    
- Foguete voa com motores
    

Apesar do mesmo método, **o comportamento é diferente**.

---

# Modificadores de Acesso

## O que são Modificadores de Acesso

São palavras-chave que controlam a **visibilidade de classes, métodos e variáveis**.

Eles ajudam no encapsulamento e definem como os elementos do programa interagem.

---

## Modificador public

Permite que o elemento seja acessível:

- De qualquer classe
    
- Em qualquer pacote
    

### Exemplo

`public class Car {
     public String model; 
}`

O atributo `model` pode ser acessado por qualquer outra classe.

---

## Modificador private

Restringe o acesso **somente à própria classe** onde o elemento foi declarado.

### Exemplo

`public class Car {      
private String color;      
private void displayColor() {
    System.out.println("Car Color: " + color);     
}
}`

Nenhuma outra classe pode acessar diretamente `color`.

---

## Modificador protected

Permite acesso:

- Dentro do mesmo pacote
    
- Por subclasses, mesmo em pacotes diferentes
    

### Exemplo

`public class Car {      
protected int year;      
protected void displayYear() {
	System.out.println("Car Year: " + year);     
} 
}`

---

## Modificador padrão (default)

Usado quando **nenhum modificador é especificado**.

### Características

- Acessível apenas dentro do mesmo pacote
    
- Não permite acesso externo
    

---

# Modificadores Sem Controle de Acesso

## Modificador static

Indica que o membro pertence à **classe**, não às instâncias.

### Características

- Compartilhado por todos os objetos
    
- Pode ser acessado sem criar um objeto
    

### Exemplo

`public class Car {      
static int numeroDeCarros = 0;
    public Car() {
	    numeroDeCarros++;     
	} 
}`

---

## Modificador final

Define que:

- Uma variável não pode ser alterada
    
- Um método não pode ser sobrescrito
    
- Uma classe não pode ser herdada
    

### Exemplo

`public final class Veiculo {      
final int velocidadeMaxima = 120;
      final void displayVelocidadeMaxima() { 
    } 
}`

---

## Modificador abstract

Usado em **classes e métodos abstratos**.

### Características

- Classes abstratas não podem ser instanciadas
    
- Métodos abstratos devem ser implementados pelas subclasses
    

### Exemplo Conceitual

Uma classe abstrata `Shape`:

- Não pode ser instanciada
    
- Define um método `draw()`
    
- Obriga as subclasses a implementarem esse método
    

---

# Importância do Encapsulamento

## Por que o Encapsulamento é Necessário

- Controla como os dados são acessados e modificados
    
- Garante o uso correto das informações
    
- Facilita manutenção e atualizações
    
- Melhora a legibilidade do código
    

---

## Manutenção de Estado

Em sistemas onde objetos mudam com frequência:

- O encapsulamento permite gerenciar estados internos
    
- O usuário interage sem conhecer a lógica interna
    
- Torna o sistema mais seguro e organizado

# Construtores em Java

## O que são Construtores

Construtores são responsáveis por **inicializar objetos**, atribuindo valores iniciais aos atributos.

### Características

- Possuem o mesmo nome da classe
    
- Não possuem tipo de retorno
    
- São chamados automaticamente na criação do objeto
    

---

## Construtor Padrão

É criado automaticamente pelo Java quando nenhum construtor é definido.

### Exemplo

`class Dog {      
String nome;      
Dog() {
    nome = "desconhecido";     
} 
}`

---

## Construtor Parametrizado

Permite passar valores no momento da criação do objeto.

### Exemplo

`class Dog { 
     String nome;     
      Dog(String dogNome) {
        nome = dogNome;     
    }
}`

---

## Construtor no-Arg

É um construtor **sem parâmetros**.

### Características

- Permite criar objetos com valores padrão
    
- Criado automaticamente se nenhum construtor for definido
    

---

## Por que Usar Construtores

- Garantem inicialização correta dos objetos
    
- Melhoram a clareza e organização do código
    
- Permitem flexibilidade com sobrecarga de construtores
    

### Boas Práticas

- Preferir construtores parametrizados
    
- Evitar lógica complexa dentro dos construtores
    
- Evitar excesso de sobrecarga

# Interface Cloneable

## O que é Cloneable

A interface `Cloneable` permite que objetos sejam **clonados**.

### Implementação

`public class Book implements Cloneable { }`

Isso indica que a classe permite a clonagem de seus objetos.

---

## Método clone()

`protected Object clone() throws CloneNotSupportedException`

### Funcionamento

- Chama o método `clone()` da superclasse
    
- Cria uma cópia do objeto
    
- Clona todos os atributos para um novo objeto
  
  
## Getters e Setters

Eles são **métodos** usados para **acessar e modificar atributos privados** de uma classe.
Isso faz parte do conceito de **encapsulamento** da Programação Orientada a Objetos.

### Em Java, a boa prática é deixar os atributos como `private` para:

- Proteger os dados.
    
- Evitar alterações indevidas.
    
- Controlar como o valor é lido ou alterado.
  
Como o atributo é `private`, **outras classes não podem acessá-lo diretamente**. É aí que entram os **getters** e **setters**.

### Geters

Serve para **ler** o valor do atributo.

```
public String getName() {
    return name;
}

public int getAge() {
    return age;
}
```

**Finalidade do getter**:

- Retornar o valor do atributo.
    
- Acesso controlado aos dados.
  
### Seters 

Serve para **modificar** o valor do atributo.

```
public void setName(String name) {
    this.name = name;
}

public void setAge(int age) {
    if (age > 0) {   // validação (opcional)
        this.age = age;
    }
}
```

**Finalidade do setter**:

- Alterar o valor do atributo.
    
- Permitir validações antes da mudança.
