# Cloud-Native

## O que é?

É uma abordagem moderna para desenvolver aplicações:

- Escaláveis
- Dinâmicas
- Preparadas para a nuvem

✔ Baseada em:

- **Microserviços**
- **Contêineres**

---

# Contêineres

## O que são?

Unidades de software que **empacotam tudo que o app precisa**:

- Código
- Bibliotecas
- Configurações

✔ Funcionam em:

- Nuvem
- Desktop
- Local

✔ Compatíveis com:

- Linux
- Windows

---

## Por que usar?

Problemas do modelo tradicional:

- Alto custo de manutenção
- Baixa escalabilidade
- Pouca portabilidade
- Difícil automação
- Uso ineficiente de recursos

✔ Contêiner resolve:

- Isolamento
- Portabilidade
- Eficiência
- Escalabilidade

---

## ✅ Benefícios

- Implantação rápida
- Redução de custos
- Melhor uso de CPU/memória
- Automação facilitada

---

## ❌ Desvantagens

- Risco de segurança (dependência do SO)
- Complexidade para gerenciar muitos contêineres

---

## Ferramentas

- **Docker:** mais popular
- **Podman:** mais seguro (sem daemon)
- **LXC:** foco em alto desempenho

---

# Docker

## O que é?

Plataforma para:

- Criar
- Executar
- Distribuir contêineres

✔ Isola aplicação da infraestrutura

---

## ✅ Benefícios

- Ambientes consistentes
- Deploy rápido
- Imagens leves e reutilizáveis
- Automação

---

## ❌ Limitações

- Não ideal para:
    - Aplicações muito críticas em segurança
    - Alto desempenho extremo
    - Sistemas monolíticos complexos

---

# Criação de um Contêiner

## Etapas

1. Criar um **Dockerfile**
2. Gerar uma **imagem**
3. Rodar um **contêiner**

---

## Exemplo de Dockerfile

FROM ubuntu:latest  
CMD ["echo", "Hello World"]

---

## Comandos básicos

docker build -t my-app:v1 .  
docker run my-app:v1  
docker ps -a

---

## Comandos importantes

- `push` → envia imagem
- `pull` → baixa imagem

---

# Objetos do Docker

## Dockerfile

- Arquivo com instruções
- Comandos principais:
    - `FROM` → imagem base
    - `RUN` → executa comandos
    - `CMD` → comando padrão

---

## Docker Image

- Modelo somente leitura
- Base para criar contêiner

Formato do nome:

host/repositório:tag

---

## Docker Container

- Instância executável da imagem
- Pode:
    - Rodar
    - Parar
    - Conectar redes
    - Usar armazenamento

---

# Arquitetura do Docker

## Componentes

- **Cliente (CLI/API)**
- **Docker Host**
- **Daemon (dockerd)**
- **Registro (Docker Hub, etc.)**

---

## Fluxo

1. Cliente envia comando
2. Daemon processa
3. Cria/roda contêiner
4. Imagem armazenada no registro

---

## Registro

- Armazena imagens
- Pode ser:
    - Público
    - Privado

---

# Kubernetes

## O que é?

Sistema de **orquestração de contêineres**

---

## Funções

- Gerenciar contêineres
- Escalar automaticamente
- Reiniciar falhas
- Balancear carga

---

## Exemplo

- Aumento de usuários → cria mais contêineres
- Queda → remove contêineres

---

# Fundamentos Importantes

## Ágil

- Desenvolvimento iterativo
- Entrega rápida de valor

---

## Cliente-Servidor

- Cliente → faz requisição
- Servidor → responde

---

## Container

- Unidade isolada com tudo necessário

---

## Registro de Containers

- Armazena e distribui imagens

---

## CI/CD

- Automação de build, teste e deploy

---

## Nativo na Nuvem

- Aplicação feita para rodar na nuvem

---

## Sem Daemon

- Sem processo central (mais seguro)

---

## DevOps

- Integra desenvolvimento + operações
- Automação + colaboração

---

## Docker (conceito reforçado)

- Plataforma de contêineres

---

## Dockerfile

- Define como a imagem será criada

---

## Cliente Docker

- Interface para interagir com o Docker
- Usa API para comunicar com o daemon

---

# Resumo Final

- **Cloud-native** = apps feitos para nuvem
- **Contêineres** = portabilidade + isolamento
- **Docker** = ferramenta principal
- **Kubernetes** = orquestração
- **CI/CD + DevOps** = automação e eficiência
