# DevOps:

Representa uma mudança cultural e tecnológica que promove a colaboração entre os engenheiros de desenvolvimento e operações em todo o ciclo de vida do dev de software. Tudo se resume em simplificar processos, automatizar tarefas...

Requisitos DevOps: mudar a cultura da empresa para trabalhar em conjunto, uma nova aplicação de design, automatização para os microsservices, usar uma plataforma programável dinâmica.

DevOps pipelines: planejar, desenvolver, fases de construção, teste e implantação. Quando falamos sobre CI/CD, estamos nas fases de construção e teste do pipeline de DevOps.

# CI/CD:

Integração contínua e entrega contínua. CI garante que as alterações passem pelos testes antes de serem integradas a ramificação principal, preparando o código para implantação. CD garante uma implantação rápida e segura ao fornecer mudanças em um ambiente, se assemelhando muito a um sistema de produção.

Contínuo Implantação (CD): Entrega Contínuo é quando você a implanta em algum lugar, como um servidor de desenvolvimento, um servidor de teste... Implantação contínua: quando você avança continuamente a produção.  

Benefícios: vai ficar mais rápido tempos de reação às mudanças no código. Código reduzido e mais integração, menos riscos, alta qualidade de código e a cada pull request é uma oportunidade para revisão de código. Leva menos tempo para implantação porque tudo já foi testado e as implatanções são automaticas, sendo mais rápida.

Benefícios do CI: tempos melhores porque toda vez que você faz uma alteração no código e o envia para uma ramificação remota, o código é testado. Mesmo que você esqueça de executar os testes, a ferramenta CI o testará pra você. Quando a alteração foi criada, mesmo que você esqueça de verificar se a compilação funciona, a ferramenta de CI a verificará, logo, você entrega mais rápido o aplicativo porque tudo vai ser testado e aprovado.


# Desenvolvimento orientado por testes (TDD):

Significa que seu caso de teste orienta o design e o desenvolvimento do seu código. Você escreve os casos de testes primeiro que o código. Também, depois de clonar um código, testa o código pra saber se quebrou algo ou já estava quebrado. TDD é importante porque reduz o tempo de debug, caso algo saia do esperado sobre o desenvolvimento, será alertado de maneira mais precisa, você fica mais confiante porque sabe que não terá problemas com o código quebrado em algum momento...

Fluxo de trabalho do TDD: você escreve um caso de teste que falhou, para o futuro código, em seguida, você escreve código suficiente para fazê-lo passar, agora você refatora o código para melhorá-lo. Esse é o RED GREEN REFACTOR. 


# Desenvolvimento orientado por comportamento (BBD):

Ele se concentra no comportamento do sistema conforme observado de fora para dentro. É olhar para o aplicativo e identificar como ele se comporta da mesma forma que um usuário do sistema faria. Ele é excelente para testes de integração, para verificar como todos os componentes estão trabalhando juntos.

Diferente entre ele e o TDD: no TDD você testa as funções do sistema de dentro pra fora ou de baixo para cima, está testando o funcionamento interno das funções, ou seja, usamos TDD para testes unitários. Por exemplo: o programa devolve os dados corretos? BDD prioriza o teste, portanto em um nível mais alto, considerando como um sistema deve funcionar sem se preocupar com o funcionamento interno. Por exemplo, em um site que o usuário faz uma compra, queremos saber se o carrinho de compras aparece para o cliente durante o uso, ou seja, não queremos saber qual API é, qual requisição, apenas o funcionamento.
 
Processo de teste de software:

Unit test: testa unidades ou componentes individuais de um sistema de software. O objetivo é avaliar cada unidade para saber se está funcionando.

Teste de integração: combina unidades individuais e testa em grupo, assim, pode expor falhas sobre a interação entre as unidades.

Teste do sistema: testamos o sistema de ponta a ponta com o objetivo de avaliar a conformidade do sistema e garantir que todo o sistema esteja funcionando em conjunto.
