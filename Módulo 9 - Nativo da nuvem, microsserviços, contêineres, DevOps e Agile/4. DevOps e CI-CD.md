# DevOps

## O que é?

DevOps é uma **cultura + conjunto de práticas** que integra:

- Desenvolvimento (Dev)
- Operações (Ops)

✔ Objetivo:

- Automatizar processos
- Melhorar colaboração
- Entregar software mais rápido e com qualidade

---

## Requisitos do DevOps

- Mudança cultural (times integrados)
- Design moderno de aplicações
- Automação (principalmente em microserviços)
- Uso de plataformas dinâmicas e programáveis

---

# Pipeline DevOps

## Etapas

1. Planejamento
2. Desenvolvimento
3. Build (construção)
4. Testes
5. Deploy (implantação)

✔ CI/CD atua principalmente em:

- Build
- Teste
- Deploy

---

# CI/CD

## O que é?

- **CI (Integração Contínua):** integra e testa código automaticamente
- **CD (Entrega/Implantação Contínua):** automatiza deploy

---

## CI (Integração Contínua)

- Código testado automaticamente
- Verifica:
    - Testes
    - Build
    - Integração

✔ Benefícios:

- Detecta erros cedo
- Evita código quebrado
- Acelera desenvolvimento

---

## CD (Entrega / Implantação Contínua)

### 🔹 Entrega Contínua

- Código pronto para produção
- Deploy manual

### 🔹 Implantação Contínua

- Deploy automático em produção

---

## Benefícios do CI/CD

- Deploy mais rápido
- Menor risco
- Melhor qualidade de código
- Feedback rápido
- Automação total
- Cada Pull Request vira revisão

---

# TDD (Test Driven Development)

## O que é?

Desenvolvimento guiado por testes.

✔ Você escreve:

1. Teste
2. Código
3. Refatora

---

## Ciclo TDD

- 🔴 **Red:** teste falha
- 🟢 **Green:** código faz passar
- 🔵 **Refactor:** melhora o código

---

## Benefícios

- Menos bugs
- Debug mais rápido
- Código mais confiável
- Segurança nas mudanças

---

# BDD (Behavior Driven Development)

## O que é?

Focado no **comportamento do sistema do ponto de vista do usuário**.

---

## Diferença TDD vs BDD

|TDD|BDD|
|---|---|
|Testa funções internas|Testa comportamento|
|Baixo nível|Alto nível|
|Teste unitário|Teste de integração|
|Foco técnico|Foco no usuário|

---

## Exemplo

- **TDD:** função retorna valor correto?
- **BDD:** usuário consegue comprar no site?

---

# Tipos de Teste

## 🔹 Unit Test

- Testa partes isoladas
- Ex: função

---

## 🔹 Teste de Integração

- Testa interação entre módulos

---

## 🔹 Teste de Sistema

- Teste completo (end-to-end)

---

## 🔹 Teste de Aceitação

- Valida requisitos do usuário

---

# Ferramentas DevOps

## Version Control

- Gerencia código
- Ex: Git

---

## Ferramentas de CI

- **Concourse CI**
    - Baseado em contêineres
    - Organiza pipelines
    - Integra com Docker

---

## Outras ferramentas comuns

- **Jenkins:** automação de CI/CD
- **GitHub Actions:** CI/CD integrado ao GitHub
- **GitLab CI:** pipeline completo integrado
- **CircleCI:** foco em performance
- **Travis CI:** integração simples

---

# Integração com Contêineres

- CI/CD + Docker = deploy consistente
- Testes podem rodar em ambientes isolados
- Facilita escalabilidade

---

# Fluxo típico DevOps

1. Dev faz commit
2. CI executa testes
3. Build é gerado
4. CD realiza deploy
5. Monitoramento contínuo

---

# Monitoramento e Feedback

- Logs
- Métricas
- Alertas

✔ Permite:

- Detectar falhas rapidamente
- Melhorar performance
